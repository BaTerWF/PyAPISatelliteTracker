Стек технологий:
Python 3.11 — основная версия Python, используемая в проекте. Важно: не используйте Python 3.12, так как не все библиотеки могут работать корректно.
FastAPI — для построения API.
PostgreSQL — реляционная база данных (если потребуется).
Astropy — библиотека для астрономических и астрофизических расчетов.
PEP 8 — стандарт для написания чистого и читаемого кода.
Основные принципы работы с репозиторием:
Ветка main предназначена для стабильных версий.
Не выполняйте прямые изменения в этой ветке. Для работы над задачами создавайте отдельные ветки.

Работа с ветками:
Каждая задача должна выполняться в отдельной ветке, чтобы избежать конфликтов и поддерживать контроль версий.

Pull Requests (PR):
Когда ваш функционал готов и протестирован, создайте Pull Request. Курирующий проект разработчик проверит его и примет (или отклонит) изменения.

Тестирование:
Перед созданием Pull Request убедитесь, что все тесты пройдены и функционал работает корректно.

PEP 8:
Соблюдайте стандарты кодирования согласно PEP 8.

Установка и настройка
Клонирование репозитория:

git clone https://github.com/User/PyAPISatelliteTracker.git
cd project
Создание виртуального окружения:

python3.11 -m venv venv
source venv/bin/activate  # Для Linux/MacOS
.\venv\Scripts\activate    # Для Windows

Установка зависимостей:

После активации виртуального окружения установите все зависимости:
pip install -r requirements.txt

Для запуска проекта на локальном сервере:

uvicorn main:app --reload
Работа с базой данных (PostgreSQL)
Если проект использует базу данных, подключение PostgreSQL настраивается через переменные окружения или .env файл.
Убедитесь, что в системе установлен PostgreSQL, и выполните необходимые команды для создания базы данных.

Как вносить изменения:
Создайте новую ветку для вашей задачи:
git checkout -b feature/your-feature-name
После завершения работы и тестирования создайте Pull Request в ветку main.

Требования
Все используемые зависимости проекта хранятся в файле requirements.txt. Для его обновления:

После установки новой библиотеки добавьте её в requirements.txt:
pip freeze > requirements.txt

Чтобы установить все зависимости проекта, выполните:
pip install -r requirements.txt
Подробнее о файле requirements.txt можно прочитать здесь https://www.freecodecamp.org/news/python-requirementstxt-explained/.

Рекомендации по работе
Использование PyCharm: Для удобства работы с Python рекомендуется использовать среду разработки PyCharm. Она обеспечивает поддержку виртуальных окружений и интеграцию с Git.

Тянуть изменения из main: Регулярно обновляйте вашу ветку с ветки main для получения актуальных изменений.
